# Support and Bug Reports

Use the repository issue templates for reproducible defects and feature proposals.
Support is best-effort until published release channels and supported versions are
defined.

Before reporting a bug, reproduce with the exact packaged build, check the known
issues, and reduce the active mod list where safe. Provide RimWorld and Emerald
Isle versions, operating system, DLC and mod list/order, new or existing save,
steps, expected and actual behavior, and relevant log excerpts. Remove personal
paths and secrets.

Feature ideas begin with design and milestone fit, not implementation requests.
Use the feature proposal template and expect ideas outside the active milestone to
remain deferred.

# Security Policy

## Reporting

Do not open a public issue for vulnerabilities that could expose users, files,
credentials, or systems. Use GitHub's private security-advisory feature once the
remote repository is configured. Include affected version, environment, impact,
reproduction, logs with personal paths removed, and any suggested mitigation.

Never attach private save files without confirming and minimizing their contents.

## Scope

Security concerns include unsafe file or process access, path traversal, arbitrary
code execution, untrusted deserialization, credential exposure, malicious update
paths, dependency compromise, and privacy-sensitive logging. Ordinary gameplay
bugs use the bug-report template.

## Response

The maintainer acknowledges reports when capacity allows, validates impact,
coordinates remediation and release, credits reporters with consent, and avoids
publishing exploit detail before a fix is available. Only supported versions are
guaranteed consideration once the support policy is adopted.
